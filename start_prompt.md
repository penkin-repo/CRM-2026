Промт для создания Dashboard рекламного агентства
От себя. Нужна база данные которая будет на firebase дислоцироваться. Просчитай хватитт ли бесплатного тарифа. 
Тут в папке уже лежат файлы фронтенда. 
Сервер будет использоваться versel 


Создай веб-приложение Dashboard для рекламного агентства на React + TypeScript + Tailwind CSS. Интерфейс полностью на русском языке. Все данные хранятся в localStorage. Приложение должно быть полностью функциональным, с тестовыми данными для демонстрации.

Общая архитектура
Приложение — одностраничное с вкладками в верхней навигации:

📋 Заказы (главный экран, dashboard)
👥 Клиенты
🏗️ Подрядчики
💳 Плательщики
📊 Отчёты
🕐 История
В шапке справа — кнопка «↩ Отменить» для отмены последнего действия, и кнопки экспорта/импорта JSON.

Сущности и структура данных
Заказ (Order)
id — уникальный идентификатор
date — дата заказа (по умолчанию сегодня)
clientId — ссылка на клиента
productName — наименование продукции (текст)
contractors — массив работ подрядчиков, каждый элемент:
contractorId — ссылка на подрядчика
description — описание работ
costFormula — формула или число стоимости (строка, может содержать математические выражения типа =6*3*2*450 или =5000*8+2000*15)
costValue — вычисленное числовое значение
payerId — кто оплатил (ссылка на плательщика)
paid — галочка «оплачено» (boolean)
reconciled — галочка «сверка» (boolean)
note — примечание к работе
totalSale — сумма реализации (число)
status — статус: 'active' или 'completed'
note — общее примечание к заказу
createdAt — дата создания записи
Вычисляемые поля:

Сумма затрат = сумма всех costValue подрядчиков
Прибыль = сумма реализации − сумма затрат
Рентабельность (%) = (прибыль / сумма реализации) × 100
Клиент (Client)
id — уникальный идентификатор
name — название (обязательное)
phone — телефон
contactPerson — контактное лицо
email — электронная почта
note — примечание (свободный текст)
customFields — массив дополнительных полей { label: string, value: string }, пользователь может добавлять/удалять/редактировать сколько угодно доп. полей
Подрядчик (Contractor)
id — уникальный идентификатор
name — название
Плательщик (Payer)
id — уникальный идентификатор
name — название (например: «Агентство наличные», «Клиент напрямую», «Директор»)
Вкладка «📋 Заказы» (Dashboard — главный экран)
Панель статистики
Сверху 4 карточки:

Всего заказов (с учётом фильтров)
Активных заказов
Общая реализация
Общая прибыль
Фильтры (сохраняются в localStorage после перезагрузки!)
Статус: все / активные / выполненные (кнопки-переключатели)
Период по датам: два поля «с» и «по» (date picker)
Месяц: выпадающий список (Январь-Декабрь + год)
Поиск по тексту: поле ввода, ищет по всей информации заказа — наименование продукции, имя клиента, имена подрядчиков, примечания, описания работ
Сортировка: по дате (новые/старые), по сумме реализации, по прибыли
Кнопка сброса всех фильтров
Таблица заказов
Колонки: Дата, Клиент, Продукция, Затраты, Реализация, Прибыль, Рентабельность (%), Статус (чекбокс для быстрого переключения), Действия (просмотр, редактировать, дублировать, удалить).

Строки окрашены в зависимости от статуса: зелёный для выполненных, белый для активных.

Кнопка «+ Новый заказ»
Открывает модальное окно создания.

Модальное окно создания/редактирования заказа
Дата — автоматически сегодняшняя
Клиент — поле поиска с выпадающим списком (SearchSelect компонент). Так как клиентов может быть много, нужен поиск в реальном времени. Выпадающий список должен рендериться через portal (createPortal), чтобы не обрезался внутри модального окна
Наименование продукции — текстовое поле
Сумма реализации — числовое поле
Примечание — textarea
Блок подрядчиков — таблица в стиле Excel:

Кнопка «+ Добавить подрядчика»
Каждая строка содержит:
Подрядчик (SearchSelect с поиском)
Описание работ (текст)
Стоимость (поле ввода, поддерживает формулы: если начинается с =, то вычисляется математическое выражение, например =1*200+(200+9888). Рядом показывается вычисленный результат)
Плательщик (SearchSelect с поиском)
Галочка «Оплачено»
Галочка «Сверка»
Примечание
Кнопка удаления строки
Снизу — итого затрат, прибыль, рентабельность (вычисляется в реальном времени).

Модальное окно просмотра заказа
Показывает всю информацию в read-only формате. Кнопки: «Редактировать», «Дублировать», «Закрыть».

Дублирование заказа
Создаёт копию заказа с сегодняшней датой, статус «активный», все галочки оплаты и сверки сбрасываются.

Вкладка «👥 Клиенты»
Поиск
Поле поиска в реальном времени — ищет по всей информации клиента (название, телефон, контактное лицо, email, примечание, дополнительные поля).

Список клиентов
Отображается сеткой карточек. Каждая карточка показывает только название клиента. При клике открывается модальное окно.

Модальное окно клиента (просмотр)
Название клиента крупно
Раскрывающийся блок с подробной информацией (телефон, контактное лицо, email, примечание, доп. поля)
Кнопка «Показать заказы» — показывает список всех заказов этого клиента, каждый заказ кликабельный для просмотра деталей
Кнопки: «Редактировать», «Удалить», «Закрыть»
Модальное окно создания/редактирования клиента
Название (обязательное)
Телефон, контактное лицо, email, примечание
Блок дополнительных полей: можно добавить сколько угодно пар «Название поля» — «Значение». Каждое можно удалить.
Вкладка «🏗️ Подрядчики»
Поиск по имени
Список подрядчиков
Кнопка «+ Добавить» — модальное окно с одним полем «Название»
Редактирование и удаление
Вкладка «💳 Плательщики»
Аналогично подрядчикам: список, поиск, добавление/редактирование/удаление через модалки
Содержит только поле «Название»
Вкладка «📊 Отчёты»
Внутри вкладки — под-вкладки (табы), так как в будущем будут добавляться новые отчёты:

Под-вкладка «📅 Месячный отчёт»
Выбор месяца и года
Поле «% зарплаты от прибыли» (по умолчанию 60%)
Поиск по контексту — текстовое поле для поиска по содержимому заказов
Карточки статистики: количество заказов, общая реализация, общие затраты, прибыль, зарплата сотрудника
Таблица заказов за выбранный месяц
Строки кликабельные — открывают модалку с деталями заказа
Под-вкладка «👤 По клиенту»
Фильтр: выбор клиента (SearchSelect)
Фильтр: период (с — по)
Поиск по контексту — текстовое поле
Карточки статистики за период
Таблица заказов выбранного клиента
Строки кликабельные — открывают модалку с полными деталями заказа
Кнопка «Дублировать» в модалке
Под-вкладка «🏗️ По подрядчику»
Фильтр: выбор подрядчика (SearchSelect)
Фильтр: период (с — по)
Поиск по контексту — текстовое поле
Карточки: итого, оплачено, не оплачено, сверено
Таблица со строками по каждой работе подрядчика, колонки:
Дата, Клиент, Продукция, Описание работ, Стоимость
Плательщик — выпадающий список, редактируемый прямо из отчёта
Оплачено — чекбокс, переключается прямо из отчёта
Сверка — чекбокс, переключается прямо из отчёта
Кнопка 👁 для просмотра полного заказа
Вкладка «🕐 История»
Система истории изменений
Каждое действие пользователя записывается с описанием и снапшотом всех данных
Записываются: создание/редактирование/удаление заказов, клиентов, подрядчиков, плательщиков; изменения из отчётов; импорт данных
Хранится до 50 последних записей
Каждая запись содержит: временную метку, описание действия, полный снапшот данных
Отображение
Список всех действий в обратном хронологическом порядке
У каждой записи кнопка «↩ Восстановить» — откатывает ВСЕ данные на этот момент
При восстановлении показывается тост-уведомление
Кнопка «↩ Отменить» в шапке
Всегда доступна
Показывает badge с количеством действий в истории
При нажатии — откатывает последнее действие
Экспорт/Импорт
Кнопка «📥 JSON» — скачивает все данные (заказы, клиенты, подрядчики, плательщики) в JSON-файл
Кнопка «📤 Импорт» — загружает JSON-файл и заменяет все данные (с записью в историю)
Технические требования
Модальные окна
Все модалки закрываются при клике на затемнённый фон (overlay)
Анимация появления (fadeIn + slideIn)
Корректная прокрутка при длинном содержимом
SearchSelect (компонент выбора с поиском)
Поле ввода для фильтрации списка
Выпадающий список рендерится через createPortal в document.body с position: fixed, чтобы не обрезался внутри модальных окон
Позиция пересчитывается при скролле и ресайзе
Закрывается при клике вне компонента
Формулы в поле стоимости
Если значение начинается с =, парсить как математическое выражение
Поддержка: +, -, *, /, скобки ()
Вычислять безопасно (без eval на произвольном коде — использовать Function с проверкой на допустимые символы)
Показывать вычисленный результат рядом с формулой
Фильтры Dashboard
Все настройки фильтров сохраняются в localStorage
При перезагрузке страницы фильтры восстанавливаются
Цветовая схема
Основной фон: серо-голубой (#f1f5f9)
Акцентный цвет: синий (#3b82f6)
Успех/выполнено: зелёный
Прибыль положительная: зелёный, отрицательная: красный
Карточки статистики: синий, зелёный, фиолетовый, оранжевый градиенты
Тестовые данные
При первом запуске (пустой localStorage) загрузить демо-данные:

10 клиентов с разными данными
8 подрядчиков
5 плательщиков
15-20 заказов за последние 3 месяца (часть активных, часть выполненных), с формулами, несколькими подрядчиками, примечаниями
Стек технологий
React 18+ с TypeScript
Tailwind CSS для стилей
Vite для сборки
localStorage для хранения данных
Никаких внешних библиотек для UI (всё с нуля на Tailwind)
uuid для генерации идентификаторов
